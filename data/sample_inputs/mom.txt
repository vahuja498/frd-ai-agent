MINUTES OF MEETING — Customer Portal Project
Date: 2025-01-15

DECISIONS MADE:

1. Project Goal
   - Build a self-service web portal for enterprise clients
   - Reduce support call volume by allowing self-service for top 5 support categories

2. Authentication
   - SSO via SAML 2.0 or OAuth 2.0 (MANDATORY — client IT policy)
   - Support Azure AD and Okta as identity providers
   - No standalone username/password authentication

3. Account Dashboard
   - Show: subscription tier, API call usage, active user count, data storage used, renewal date
   - Data refresh: daily for MVP; real-time in Phase 2
   - Source: internal usage database (TBD — James to confirm schema)

4. Invoice Management
   - Source: Stripe billing system
   - Display invoice history with PDF download
   - No invoice creation in this portal — display only
   - James to confirm Stripe API access credentials with Mark's team

5. Support Tickets
   - Integration with Zendesk API
   - Clients can: raise new ticket, view ticket list, view ticket detail, add comments
   - Status updates should trigger email notification
   - Priya to document Zendesk API integration requirements separately

6. User Management
   - Client Admin can: invite team members, assign roles, deactivate users
   - Roles defined: Client Admin, Client Team Member, Platform Super-Admin
   - Platform Super-Admin: view all clients, impersonate user (for support), manage platform settings

7. Email Notifications
   - Trigger 1: New invoice available
   - Trigger 2: Support ticket status changed
   - Trigger 3: Subscription renewal reminder — 30 days before renewal date
   - Email provider: TBD (James to recommend — SendGrid or AWS SES)

8. Non-Functional
   - GDPR compliance required (EU clients)
   - 500 concurrent users at launch; 2000 within 18 months
   - Responsive web only — no native mobile apps this phase
   - Target go-live: end of Q3 (approx 20 weeks)
